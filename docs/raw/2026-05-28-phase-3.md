## Goal

`f1_predict_pit_strategy(session_key, driver_number)` returns a sane pit-stop recommendation on a recorded 2025 race: ordered stop laps, compound sequence, and an expected finish-position delta. Backed by a tyre-degradation model fit to OpenF1 lap data and an undercut model that accounts for pit-lane loss + fresh-tyre delta.

Scope: **6 F1 RAW + 5 F1 INTEL tools** —

RAW: `f1_get_sessions`, `f1_get_drivers`, `f1_get_lap_times`, `f1_get_standings`, `f1_get_race_results`, `f1_get_weather`.

INTEL: `f1_tyre_degradation`, `f1_undercut_window`, `f1_head_to_head_pace`, `f1_weather_strategy_impact`, `f1_predict_pit_strategy` (flagship).

## Step 6 — Models: tyre degradation, undercut, pit strategy

**Goal:** Pure functions that compose lap-time data into a strategy recommendation.

**Files:**

- `src/sportiq/f1/models/__init__.py` (new, empty).
- `src/sportiq/f1/models/tyre_deg.py` (new) — `fit_degradation(laps: list[dict], compound: str) -> {intercept, slope, residual_std, sample_count}`. Filters laps to one compound + driver + valid lap times (drops in/out laps and SC laps), runs `numpy.polyfit(degree=1)` on lap-time vs tyre-life. Returns the linear model. **No scipy needed.**
- `src/sportiq/f1/models/undercut.py` (new) — `undercut_window(driver_pace_s, target_pace_s, pit_loss_s, fresh_tyre_delta_s, gap_to_target_s) -> {laps_to_clear, viable, marginal}`. Pure arithmetic — boundary cases tested.
- `src/sportiq/f1/models/pit_strategy.py` (new) — `predict(laps, stints, weather, current_lap, total_laps) -> {stop_laps: [int], compound_sequence: [str], expected_finish_position: int | None, confidence: float}`. Composes `tyre_deg` (per-compound rate) + `undercut` (vs nearest rivals) + a simple race-time simulator that walks lap-by-lap and chooses stops where projected lap-time savings exceed pit loss.
- `tests/unit/test_tyre_deg.py` (new) — synthetic increasing-laps data → positive slope; flat data → ~0 slope; outliers filtered.
- `tests/unit/test_undercut.py` (new) — clean clear / marginal / no-clear / no-gap edge cases.
- `tests/unit/test_pit_strategy.py` (new) — synthetic 50-lap race, one driver: assert exactly one stop on a 1-stop strategy with SOFT→MEDIUM, two stops on hot weather with HARD scarcity.

**Verify:** Unit tests green; `predict()` on a synthetic race returns a deterministic, plausible `stop_laps` list.

## Open questions (deliberate, called out for the next session)

- **session_key registry for `fastf1_local`.** OpenF1's `session_key` doesn't map directly to fastf1's `(year, round)` pair. Either build a small mapping table during Phase 3 (cache OpenF1's `/sessions` once per year) or accept that the fastf1 fallback only kicks in when the chain caller provides `year` + `round` explicitly. Decide during Step 4.
- **Tyre constants calibration.** `tyres.py` values are seeded from public references; the real 2025 numbers shift weekly per FP1 data. Phase 3 ships with static seeds; per-race auto-calibration (fit fresh constants from the last 3 race weekends) is a Phase 3.1 follow-up.
- **Pit loss per circuit.** Some circuits have much longer pit lanes (Monaco vs Bahrain). Phase 3 should seed a `data/circuits.py` with `pit_loss_s` per circuit; values are public race-engineer references.
- **`f1_predict_pit_strategy` for sessions in progress.** Until a race finishes, the model doesn't know `total_laps`. Accept it as a tool arg (`total_laps: int`) or pull it from `f1_sessions_chain` metadata? The chain has it.
- **Sprint weekends.** Sprint races have different points + tyre constraints. Phase 3 treats them as a separate session_type but does NOT yet add a sprint-specific strategy. Phase 3.1.
